### 黑名单格式示例

- **取反格式，添加后其它非 `NOT:` 开头的规则将会跳过**: `NOT:https://example.com/*`
- **正则格式**：`/^https:\/\/example\.com\/.*$/`
- **通配符格式**：`https://example.com/*`
- **纯文本格式**：`https://example.com`

#### 常见匹配模式示例
- 文件类：  
`/\.(zip|rar|7z|exe|msi|md|pdf|docx?|xlsx?|pptx?|apk|dmg|iso)(\?.*)?$/`

- 图片类：  
`/\.(jpg|jpeg|avif|png|svg|ico|webp|gif)(\?.*)?$/`

- 视频类：  
`/\.(mp4|mkv|rmvb|rm|avi|ts|mov|flv)(\?.*)?$/`

- 音频类：  
`/\.(mp3|flac|ogg|wav|aac)(\?.*)?$/`

- 下载链接类：  
`/^(magnet:\?xt=urn:[a-z0-9]+:[a-f0-9]{32,40}.*|https?:\/\/[^\s]+\/dl[a-zA-Z0-9\/%+=_-]+)$/`

# 常见问题

### 双击预览是否会影响网页原有单击和拖拽动作？
不会。

### 能否设置弹窗始终置顶？  
由于浏览器扩展的限制，插件无法直接实现此功能。  
你可以使用第三方软件，如 **PowerToys - Always On Top** 来实现。  
另外，记得在设置页面中取消勾选 **“当原页面获得焦点时关闭弹窗”**。

### 为什么预览时会触发其他动作？  
请尝试禁用其他插件测试是否仍会出现。  
如果你使用的是 Edge，请检查 **`edge://settings/superDragDrop`**。  
若问题仍存在，请提交 issue 反馈。

### 为什么商店上的插件版本不是最新的？  
由于审核流程，更新可能会延迟约 7 个工作日。

### 能否自定义预览窗口的外观？  
预览窗口基于浏览器原生窗口，暂不支持自定义样式。  
Firefox 用户可尝试通过 `userChrome.css` 进行调整。

### 能否在使用拖拽预览时保留页面原生的拖拽功能？  
你可以在设置中启用 **“仅在拖拽到空白区域时响应”** 选项，以保留网页原生的拖拽行为。


### 链接支持情况

| 链接位置                | 是否支持 |
|--------------------------|------------|
| 普通网页             | ✅ 支持     |
| iframe 内部        | ✅ 支持     |
| 开放的 shadow root 内部 | ✅ 支持  |
| 关闭的 shadow root 内部 | ⚠️ 只能通过右键菜单预览链接 |
| canvas 内部 | ❌ 不支持   |

# 隐私与数据处理说明

Peek Pop 的设计原则是：**所有功能均在用户本机完成。**

它不会向任何地方发送用户数据，不会收集分析信息或遥测数据，不会在后台请求外部服务器，也不依赖任何远程 API 或云端服务。

所有用户偏好设置、保存的数据以及运行过程中的临时状态，仅存储于浏览器的本地扩展存储中，或仅在当前运行内存中存在。

目前本扩展没有任何后端服务参与。

这意味着：

* 不会上传浏览内容
* 不会传输网页文本信息
* 不会记录或上报鼠标、键盘行为
* 不会收集任何个人信息
* 不需要账号或登录
* 不存在任何隐藏的后台网络通信

## 为什么扩展需要较多权限？

与一些仅提供简单界面功能的扩展相比，Peek Pop 所申请的权限看起来会更广一些。

这是因为 Peek Pop 需要具备：

* 与网页进行更深层的页面交互
* 在支持的网站中注入脚本
* 获取标签页及页面上下文状态
* 在本地持久化保存用户配置和运行状态

这些权限仅用于实现扩展功能本身，**不会被用于追踪、遥测或任何形式的数据收集。**
